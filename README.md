# herramienta_repaso
Aplicación para repasar contenidos trabajados en el aula. 


🌱 Proyecto "Efecto Protegé": Simulador de Estudiante Virtual con IA1. Introducción y Fundamentación PedagógicaEsta aplicación web es un simulador conversacional impulsado por Inteligencia Artificial (Gemini de Google) diseñado específicamente para el ámbito educativo (Secundaria y FP Básica).A diferencia de los usos tradicionales de la IA en el aula (donde el alumno pregunta y la máquina responde como un oráculo), esta herramienta invierte los roles basándose en el Efecto Protegé: el alumno adopta el rol de docente y la IA actúa como un estudiante virtual con dudas.El objetivo pedagógico principal no es repasar mecánicamente un texto, sino forzar la argumentación científica y el cambio conceptual. El simulador está programado para no dar respuestas, dudar de las explicaciones superficiales, exigir ejemplos de la vida real y, lo más importante, presentar concepciones alternativas (errores comunes) que el alumnado real debe diagnosticar y desmontar.2. Arquitectura y Funcionalidades PrincipalesLa aplicación está construida en Python utilizando Streamlit (para la interfaz web) y la API oficial google-genai. Se divide en dos capas principales:A. Capa del Alumnado (Frontend DUA)Interfaz accesible y libre de distracciones: Diseñada bajo los principios del Diseño Universal para el Aprendizaje (DUA). Al entrar, el alumno solo ve instrucciones claras y botones de inicio rápido.Menú de configuración: El alumno selecciona su asignatura, el tema a repasar, el idioma de interacción (Castellano/Valenciano) y el nivel de profundidad de las dudas del "estudiante virtual" (Básico, Intermedio, Avanzado).Anonimato total: No requiere registro ni cuentas de Google personales, garantizando el cumplimiento de la normativa de protección de datos.B. Capa Docente (Backend y Gestión de Contenidos)El docente no necesita programar para añadir nuevos temas. La aplicación lee automáticamente una estructura de carpetas en GitHub:Lectura automática de apuntes: El profesor sube un archivo .pdf en la carpeta correspondiente (ej. apuntes/Biologia_3_ESO/Ecosistemas.pdf). El código extrae el texto de forma invisible y lo usa para limitar la base de conocimiento de la IA, evitando que el simulador invente información fuera del currículo ("alucinaciones").Inyección oculta de Concepciones Erróneas: El docente puede subir un archivo de texto plano con el sufijo _errores.txt (ej. Ecosistemas_errores.txt) detallando las ideas previas típicas del alumnado. El sistema lee este archivo en la sombra y obliga a la IA a asumir esos errores como propios. El alumno nunca ve este archivo, preservando el conflicto cognitivo.3. Dinámica de Interacción: ¿Qué tipo de preguntas hace el simulador?El Prompt del sistema (las instrucciones maestras de la IA) obliga al simulador a seguir unas reglas inquebrantables:Nunca da la respuesta correcta: Si el alumno se atasca y le pide la solución, el simulador responde: "Creo que así no aprendería bien. Prefiero que me lo expliques tú paso a paso".Rechazo a la memorización: Si detecta que el alumno está recitando definiciones literales de un libro, le pedirá que use sus propias palabras o que le ponga una analogía de la vida real (ej. comparar un circuito eléctrico con un circuito de agua).Progresión Cognitiva: Las dudas evolucionan. Empieza con la comprensión literal, pasa a relaciones causales ("¿Qué pasa si aumento el voltaje?") y termina en transferencia ("¿Cómo aplicamos esto al proyecto de renaturalización del patio?").Generación de errores verosímiles: Si no hay un archivo de errores predefinido, la IA simula confusiones comunes: generalizaciones excesivas, simplificaciones o interpretaciones causales erróneas.4. Sistema de Evaluación MetacognitivaCuando el alumno da por terminada la sesión (pulsando el botón "Terminar y Evaluar"), el simulador ejecuta una fase de cierre estructurada en tres pasos, ideal para recoger evidencias formativas y subirlas al Entorno Virtual de Aprendizaje (Aules / Teams):Síntesis: El simulador resume lo que ha entendido gracias a la explicación del alumno.Cuestionario Metacognitivo: Realiza 3 preguntas directas al alumno sobre su propio proceso de aprendizaje (¿Qué te ha costado más explicar?, etc.).Rúbrica Formativa (Evidencias Literales): Genera una tabla de evaluación con Criterios, Nivel de Logro y, fundamental, citas literales de la conversación que demuestran ese logro.Nota: Si el docente introdujo el archivo _errores.txt, la rúbrica incluirá un ítem específico evaluando si el alumno logró identificar y desmontar esa concepción previa concreta.5. Guía de Implementación Técnica (Para Docentes)Implementar esta herramienta en un centro educativo es totalmente gratuito y requiere tres pasos clave: GitHub (alojamiento), Gemini (motor de IA) y Streamlit (despliegue web).Paso 1: Obtener la Clave API de GeminiAcceder a Google AI Studio.Crear un nuevo proyecto y generar una API Key.(Aviso para Europa): Para evitar las restricciones geográficas de la capa gratuita en la UE, el despliegue lo haremos en servidores americanos a través de Streamlit (Paso 3).Paso 2: Preparar el Repositorio en GitHubCrea un repositorio en GitHub con la siguiente estructura de archivos:/
├── app.py                  # El código principal de la aplicación en Python
├── requirements.txt        # Dependencias (streamlit, google-genai, PyPDF2)
└── apuntes/                # Carpeta raíz de los materiales
    ├── FPB_Electricidad/
    │   ├── cuadros_electricos.pdf
    │   └── cuadros_electricos_errores.txt  # (Opcional) Concepciones previas ocultas
    └── Biologia_3_ESO/
        ├── el_suelo_y_patio.pdf
Paso 3: Despliegue en Streamlit Community CloudEntrar en share.streamlit.io e iniciar sesión vinculando la cuenta de GitHub.Pulsar en "Create App" y seleccionar el repositorio y el archivo app.py.Antes de darle a Deploy, entrar en "Advanced settings" (Ajustes avanzados).En el apartado de Secrets, pegar la clave de Google con este formato exacto:GEMINI_API_KEY = "tu_clave_alfanumerica_aqui"Guardar y pulsar Deploy.En unos minutos, la aplicación estará publicada con un enlace público (ej. mi-simulador.streamlit.app). El uso de Streamlit Cloud asegura que las peticiones se realicen desde servidores de EE. UU., otorgando a la cuenta docente una cuota masiva gratuita (1.500 peticiones diarias mediante el modelo estable gemini-2.0-flash o gemini-1.5-flash), permitiendo que toda una clase de 30 alumnos interactúe simultáneamente sin cortes ni cuellos de botella.
